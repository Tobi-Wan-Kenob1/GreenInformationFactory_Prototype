Run the factory in your browser

Drop a CSV with process measurements and run the pipeline right here — powered by Python compiled to WebAssembly, no installation, no account.

🔒 Your data never leaves your machine — everything runs locally in this tab.

1 Load the analysis engine

Downloads the scientific Python stack (~70 MB, one time — your browser caches it) and the pipeline's own modules. Nothing is installed on your computer.

2 Provide your data

A delimited text file (CSV) with numeric process variables and one target column. Semicolons, decimal commas and hh:mm:ss time columns are handled automatically.

3 Confirm what each column means

The factory suggests a role for every column — the same auto-detection the offline pipeline uses. matched by name means high confidence; guessed means please check.

4 Run the pipeline

Prepare → train & compare models → sustainability indicators → scenario sweep. Everything computes locally; a laptop handles the demo in seconds.

Expert inputs (optional) — improve the sustainability estimates

Everything below has a sensible default. Anything you leave untouched is marked as defaulted in the results and the affected charts carry a high-uncertainty label.

Energy-driver weights (assumption-based proxy — how strongly each variable drives energy use in your process)
Emission factors (convert the relative CO₂ proxy into an indicative kg-CO₂e range)
Scenario baseline (which validation row anchors the what-if sweep)

5 Results

Next station: publish your results on Zenodo — see the researcher how-to, step 6. The zip includes a provenance.json documenting every input, default and engine version.
